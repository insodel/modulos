Análisis de la complejidad algorítmica usando anotación Big-O

Este gráfico muestra cómo el tiempo de ejecución de distintos algoritmos crece a medida que aumenta el tamaño de entrada.
A medida que avanzamos en la lista de algoritmos, podemos ver cómo su complejidad afecta el rendimiento.

Un algoritmo con O(1) siempre tarda el mismo tiempo, sin importar el tamaño de los datos.
Ejemplo: Acceder al primer elemento de un array. 

                        const O1 = arr => arr[0];

Un algoritmo con O(n) recorre todos los elementos del array una vez.
Ejemplo: Calcular la suma de todos los elementos de un array.

Un algoritmo con O(n²) tiene dos bucles anidados, lo que hace que el tiempo de ejecución crezca exponencialmente.
Ejemplo: Comparar todos los pares posibles en un array.

Un algoritmo con O(log n) reduce el tamaño del problema en cada paso.
Ejemplo: Búsqueda binaria en una lista ordenada.

Un algoritmo con O(n log n) suele aparecer en algoritmos de ordenamiento eficientes.
Ejemplo: Merge Sort o Quick Sort.
